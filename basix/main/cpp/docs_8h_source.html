57 Create a lattice of points on a reference cell optionally
58 including the outer surface points.
59 
60 For a given `celltype`, this creates a set of points on a regular
61 grid which covers the cell, eg for a quadrilateral, with n=2, the
62 points are: `[0,0], [0.5,0], [1,0], [0,0.5], [0.5,0.5], [1,0.5],
63 [0,1], [0.5,1], [1,1]`. If the parameter exterior is set to false,
64 the points lying on the external boundary are omitted, in this case
65 for a quadrilateral with `n == 2`, the points are: `[0.5, 0.5]`. The
66 lattice type can be chosen as type::equispaced or type::gll. The
67 type::gll lattice has points spaced along each edge at the
68 Gauss-Lobatto-Legendre quadrature points. These are the same as
69 type::equispaced when `n < 3`.
70 
71 Args:
72  celltype: The cell type
73  n: Size in each direction. There are `n + 1` points along each edge of the cell

189 const std::string tabulate_polynomial_set = R"(
190 Tabulate the orthonormal polynomial basis, and derivatives,
191 at points on the reference cell.
192 
193 All derivatives up to the given order are computed. If derivatives
194 are not required, use `n = 0`. For example, order `n = 2` for a 2D
195 cell, will compute the basis \f$\psi, d\psi/dx, d\psi/dy, d^2
196 \psi/dx^2, d^2\psi/dxdy, d^2\psi/dy^2\f$ in that order (0, 0), (1,
197 0), (0, 1), (2, 0), (1, 1), (0 ,2).
198 
199 For an interval cell there are `nderiv + 1` derivatives, for a 2D
200 cell, there are `(nderiv + 1)(nderiv + 2)/2` derivatives, and in 3D,
201 there are `(nderiv + 1)(nderiv + 2)(nderiv + 3)/6`. The ordering is
202 'triangular' with the lower derivatives appearing first.
203 
204 Args:
205  celltype: Cell type
206  ptype: The polynomial type
207  d: Polynomial degree
208  n: Maximum derivative order. Use n = 0 for the basis only.
209  x: Points at which to evaluate the basis. The shape is (number of points, geometric dimension).
210 
211 Returns:
212  Polynomial sets, for each derivative, tabulated at points.
213  The shape is `(number of derivatives computed, number of points,
214  basis index)`.
215 
216  - The first index is the derivative. The first entry is the basis
217  itself. Derivatives are stored in triangular (2D) or tetrahedral
218  (3D) ordering, eg if `(p, q)` denotes `p` order derivative with
219  respect to `x` and `q` order derivative with respect to `y`, [0] ->
220  (0, 0), [1] -> (1, 0), [2] -> (0, 1), [3] -> (2, 0), [4] -> (1, 1),
221  [5] -> (0, 2), [6] -> (3, 0),...
222  The function basix::indexing::idx maps tuples `(p, q, r)` to the
223  array index.
224 
225  - The second index is the point, with index `i` corresponding to the
226  point in row `i` of @p x.
227 
228  - The third index is the basis function index.
229  TODO: Does the order for the third index need to be documented?
230 )";
